## 발견 내역

### [경고] Pydantic v2 — class-based Config 사용 Deprecated

- **파일**: `app/core/config.py` (기존 파일, 이번 작업 범위 외)
- **문제**: `class Config: env_file = ".env"` 형식은 Pydantic v2에서 Deprecated. 런타임 경고가 테스트 실행 시 출력됨.
- **수정 여부**: 이번 작업 범위가 아닌 기존 파일이므로 수정하지 않음.
- **권고**: `model_config = SettingsConfigDict(env_file=".env")` 방식으로 전환 필요.

### [참고] ocr_service.py — 미사용 import

- **파일**: `app/services/ocr_service.py`
- **문제**: `import io` 가 선언되어 있으나 함수 내에서 사용되지 않음. 원본 명세에 포함된 내용이므로 그대로 유지.
- **수정 여부**: 명세 제공 코드 그대로 생성. 향후 리팩토링 시 제거 권고.

### [참고] upload.py — validate_upload 반환값 미활용

- **파일**: `app/api/v1/endpoints/upload.py`
- **문제**: `validate_upload(file)` 는 `bytes`를 반환하지만 `upload_screenshot` 핸들러에서 반환값을 변수에 받지 않음. 파일 내용은 이후 `ocr_service.extract_text(file)` 에서 다시 읽으므로 동작에는 문제없음.
- **수정 여부**: 로직 변경 없이 원본 명세 유지. 향후 `content = await validate_upload(file)` 로 받아 중복 읽기를 줄이는 리팩토링 가능.

## 구조 개선 제안

1. **`validate_upload` 반환값 활용**: 현재 `upload_screenshot`에서 파일을 두 번 읽는 구조 (validate_upload 내 1회 + extract_text 내 1회). `validate_upload`의 반환값 `content`를 `extract_text`에 직접 넘기도록 인터페이스를 바꾸면 I/O를 줄일 수 있음.

2. **`ocr_service._reader` 싱글턴 초기화 시점**: 현재 첫 요청 시 `easyocr.Reader`를 초기화하므로 첫 번째 요청 응답시간이 길 수 있음. FastAPI `lifespan` 이벤트에서 미리 로드하는 방식 검토 권고.

3. **`db_service.save_snapshot` 트랜잭션 단위**: 현재 멤버 하나당 `db.commit()`을 호출함. 업로드 엔드포인트에서 멤버가 여럿일 경우 중간 실패 시 일부만 저장될 수 있음. 엔드포인트 레벨에서 트랜잭션을 관리하는 구조 검토 권고.
